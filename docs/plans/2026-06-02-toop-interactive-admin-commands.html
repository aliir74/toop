Implementation Plan · 2026-06-02

Interactive (button-driven) admin commands

Planning 0 / 0 tasks 0 / 0 steps TDD

Architecture

Convert seven typed-args admin commands to inline-keyboard flows, mirroring the existing /rename pattern: a CommandHandler with no args renders InlineKeyboardMarkup of players/contacts/durations (callback_data carries short prefixes + ids), and per-prefix CallbackQueryHandlers resolve each tap. Single-step picks (remove, disable, enable, lock_in) act immediately; chained picks (pause, link) edit the message into a second keyboard; /add_player stashes the picked contact in context.user_data and consumes the next DM via a group=2 MessageHandler. Every typed one-shot form is preserved as a fallback. No new CommandHandlers; +9 CallbackQueryHandlers and +1 MessageHandler bring add_handler.call_count from 31 to 41.
