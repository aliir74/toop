Voting-pool control + accountless ghost players

Planning 0 / 0 tasks 0 / 0 steps TDD

Architecture

Two features for the Toop volleyball bot. (A) The 🤷 "Don't know" tap, currently discarded, becomes a per-pair aggregate counter; players who accumulate a high don't-know rate (nobody knows them well enough to rate) get flagged to the admin via a daily DM and can be pulled from the rating pool either on a timer or a manual toggle. (B) Accountless "ghost" players get a synthetic negative id so others can vote on them before they join Telegram, and a /link_player command later merges every vote, rating, RSVP and attendance row onto their real account. Both features preserve the privacy invariant: vote_aggregates never stores voter identity.
